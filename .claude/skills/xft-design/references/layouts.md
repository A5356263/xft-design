# Layout Rules

本文件定义业务后台页面的页面级布局规则。生成页面时，必须先读取本文件，再选择组件和生成内容。

说明：
- `assets/shells/admin-side-shell.html` 负责完整后台壳子。
- `assets/page-blocks.html` 负责可复制的页面主体结构片段。
- `references/checklist.md` 负责最终验收门。
- 本文件只定义壳子内部，尤其是 `PageContent` 内的布局原则和页面类型结构。

本文件在完整生成链中的位置是：

```text
页面类型路由
→ 框架路由
→ 复制 shell seed
→ 读取本文件确定 PageContent 布局
→ 再去选页面主体结构片段
→ 最后进入 checklist 验收
```

## 页面框架总原则

所有后台业务页面默认使用统一页面打开导航框架。

统一框架由四个核心区域组成：

```text
顶部导航 TopNav
左侧导航 SideNav
多页签 WebTabs
页面内容区 PageContent
```

标准结构：

```html
<div class="app-shell">
  <header class="top-nav"></header>
  <div class="main-frame">
    <aside class="side-nav"></aside>
    <main class="work-area">
      <nav class="web-tabs"></nav>
      <section class="page-content"></section>
    </main>
  </div>
</div>
```

必须先生成这个框架，再将业务页面主体放入 `page-content > .page-content-container`。

本文件回答的是“页面内容区应该长成什么布局”，不回答“最终复制哪个 HTML 起点”，也不回答“最终是否通过验收”。

## 页面打开导航样式框架

页面打开后，顶部导航下面展示的是多页签。多页签不是页面内部 Tab，而是当前已打开页面的切换入口。

正确结构：

```text
顶部导航
├── 系统级入口 / 全局搜索 / 用户信息
下方工作区
├── 左侧导航
└── 右侧工作区域
    ├── 多页签
    └── 页面内容
```

错误结构：

```text
顶部导航
└── 页面内容
```

错误原因：缺少多页签区域。

错误结构：

```text
顶部导航
└── 页面内容
    └── 多页签
```

错误原因：多页签被放入页面内容区内部。

错误结构：

```text
顶部导航
├── 多页签
└── 左侧导航和内容混排
```

错误原因：左侧导航和工作区域层级不清晰。

## 顶部导航规则

顶部导航用于系统级导航，不用于页面级业务内容。

顶部导航可以包含：

- 系统名称
- 产品或模块入口
- 全局搜索
- 消息
- 帮助
- 用户信息
- 全局操作

顶部导航不得包含：

- 当前表格页筛选条件
- 当前表单提交按钮
- 当前详情页编辑按钮
- 当前页面的局部 Tab
- 当前页面的业务分组

顶部导航固定高度 48px。顶部导航颜色从左到右渐变为 #3C8AFF → #4255FF。顶部导航横向铺满页面。

## 左侧导航规则

左侧导航用于模块级、页面级导航。

左侧导航位于顶部导航下方，页面内容左侧。

左侧导航固定宽度 188px，同一套系统内必须保持一致。

左侧导航仅有一级导航和二级导航，不存在三级导航。

左侧导航必须体现：

- 一级菜单（描边 icon + 导航文案）
- 二级菜单（仅导航文案，无 icon）
- 展开状态
- 选中状态

### 一级导航强制样式

一级导航必须展示为：

```text
描边 icon + 导航文案
```

正确示例：

```html
<div class="side-nav-item level-1 is-expanded">
  <svg class="nav-icon" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="1.8">
    ...
  </svg>
  <span class="nav-label">业务管理</span>
  <svg class="nav-arrow" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="2">
    <path d="M6 9l6 6 6-6"/>
  </svg>
</div>
```

禁止示例（一级导航缺少 icon）：

```html
<div class="side-nav-item level-1">
  <span class="nav-label">业务管理</span>
</div>
```

### 二级导航强制样式

二级导航仅展示文案，不展示 icon：

```text
导航文案
```

正确示例：

```html
<div class="side-nav-item level-2 is-active">
  <span class="nav-label">订单管理</span>
</div>
```

禁止示例（二级导航展示了 icon）：

```html
<div class="side-nav-item level-2">
  <svg class="nav-icon" viewBox="0 0 24 24" fill="none" stroke="currentColor" stroke-width="1.8">
    ...
  </svg>
  <span class="nav-label">订单管理</span>
</div>
```

禁止原因：二级导航不应展示 icon，仅展示文案。

### 左侧导航层级

一级导航用于模块，展示描边 icon + 文案。

二级导航用于页面，仅展示文案。

不存在三级导航。左侧导航最多只有两级。

左侧导航中当前页面对应的菜单项必须为选中态。选中态需要与多页签当前选中页保持一致。

## 多页签规则

多页签用于页面打开管理。

多页签必须位于顶部导航下方、页面内容上方。

多页签用于展示：

- 首页
- 当前页面
- 其他已打开页面
- 可关闭页签
- 溢出页签
- 上一页 / 下一页 / 更多操作

多页签不是内容区域中的 Tabs。不要用多页签承载表单分组、详情分区或筛选切换。

### 多页签结构

多页签由以下元素组成：

1. 已选择标签页
2. 页签名称
3. 页签关闭图标
4. 未选择页签
5. 分割线
6. 上一页 / 下一页图标
7. 更多页签按钮

### 多页签状态

多页签必须支持：

- 默认态
- 悬浮态
- 选中态
- 点击态
- 溢出态
- 更多菜单态

当前页面必须在多页签中有选中态。

### 多页签尺寸

未选中页签默认宽度约为 128px。

选中页签默认宽度约为 130px。

选中页签文字区域宽度约为 82px，超出文字使用省略号。

页签之间使用 1px 分割线，分割线高度约为 16px。

## Overlay 承载决策规则

当需求只涉及弹窗、抽屉、确认框等覆盖层时，应优先判断为 Page Overlay，而不是完整页面生成。

### 独立 Overlay

适用于：

- 删除确认弹窗；
- 提交确认弹窗；
- 简单编辑弹窗；
- 独立展示的抽屉或确认框；
- 用户没有要求放入某个具体页面上下文的弹窗。

处理方式：

- 使用 `blank-shell.html` 作为中性画布；
- 从 `overlays.html` 选择 `Modal / Drawer / ConfirmDialog`；
- 不生成 `TopNav`、`SideNav`、`WebTabs`；
- 不重构完整页面。

### 页面内 Overlay

适用于：

- 在列表页打开详情抽屉；
- 在表格行点击后打开编辑弹窗；
- 在详情页中发起删除、撤销、作废确认；
- 用户明确说明弹窗发生在某个后台页面中。

处理方式：

- 使用 `admin-side-shell.html` 作为弱背景；
- 背景页面只保留必要上下文；
- 不重构宿主页面；
- overlay 必须挂载到顶层 `overlay-root`；
- `overlay-root` 不得放入 `page-content` 或 `page-content-container` 内。

### 禁止行为

- 不得因为弹窗需求重新生成完整页面；
- 不得修改无关导航、菜单、表格结构；
- 不得把 overlay 结构写进 page block；
- 不得把 `overlay-root` 嵌套在滚动容器内。

更多按钮区域和上一页、下一页按钮区域位于多页签右侧。

### 多页签溢出

当页签数量超过当前宽度时：

- 不允许换行。
- 不允许挤压到不可读。
- 必须展示上一页、下一页、更多按钮。
- 隐藏页签进入更多菜单。
- 更多菜单可展示未显示页签。
- 更多菜单底部提供关闭全部。

### 多页签交互

点击页签时切换页面。

点击关闭图标时关闭页签。

点击上一页 / 下一页时横向切换页签展示范围。

点击更多按钮时展示更多页签菜单。

右键点击页签时，可展示关闭当前页、关闭其他页、关闭全部菜单。

## 页面内容区规则

页面内容区是当前业务页面主体区域。

页面内容区必须位于多页签下方。页面内容区必须有一个大背景容器 `#ffffff`（白色），所有页面内容（页面标题、筛选区、表格、表单、详情等）必须在此白色容器中展示，不允许有裸露在容器外的内容。

页面内容区白色容器内边距建议为 24px。

页面内容区可以包含：

- 页面标题
- 页面说明
- 操作按钮
- 筛选区
- 表格
- 表单
- 详情信息
- 设置分组
- 图表
- 卡片
- 时间线
- 关联列表

页面内容区不得包含：

- 顶部导航
- 左侧导航
- 页面打开多页签
- 重复的系统级导航

## 页面类型布局

### 首页

首页用于展示系统概览和高频入口。

首页内容区建议结构：

```text
PageHeader
OverviewCards
QuickAccess
TodoList
ChartPanel
RecentList
```

首页不要做成单纯表格页。首页可以使用卡片、统计数据、快捷入口、趋势图、待办等模块。

### 表格页

表格页用于查询、管理和操作数据。

表格页内容区标准结构：

```text
PageHeader
FilterBar
TableToolbar
DataTable
Pagination
```

表格页必须有数据表格。筛选条件较多时，筛选区可支持展开收起。

表格页禁用：

- 只用卡片列表替代表格，除非用户明确要求卡片列表。
- 将筛选条件放入顶部导航。
- 将批量操作放到顶部导航。
- 页面内容直接贴在顶部导航下方。

### 提单页

提单页用于创建新业务单据或提交申请。

提单页内容区标准结构：

```text
PageHeader
FormPage
FormSection
FormField
AttachmentArea
ActionBar
```

字段少于 8 个时，可以使用基础表单。

字段超过 10 个时，必须使用分组表单。

存在明确步骤时，使用分步表单。

提单页底部必须有提交和取消操作。

### 详情页

详情页用于展示单据、客户、订单、配置等对象的只读信息。

详情页内容区标准结构：

```text
PageHeader
StatusSummary
DetailLayout
DescriptionList
RelatedTable
Timeline
ActionBar
```

字段少于 8 个时，可以使用单列详情。

字段 9 到 20 个时，使用双列详情。

存在关联数据时，使用关联表格。

存在流程记录时，使用时间线。

### 编辑页

编辑页用于修改已有数据。

编辑页内容区标准结构：

```text
PageHeader
FormPage
FormSection
FormField
ActionBar
```

编辑页必须体现已有数据回填状态。

编辑页和提单页的区别是：提单页用于新建，编辑页用于修改。编辑页字段默认应有值或明确的当前配置。

### 设置页

设置页用于系统配置、业务参数、用户偏好、权限策略等。

设置页内容区标准结构：

```text
PageHeader
SettingsLayout
SettingsAnchor
SettingsSection
SettingItem
```

设置页应使用设置分组和设置项，不应默认使用普通表格作为主结构。

设置项应包含：

- 设置项标题
- 设置项说明
- 当前状态
- 控件区域

设置页适合左侧锚点 + 右侧设置分组布局。

## 弹窗、抽屉、页面承载决策

表单或详情内容的承载方式按以下规则选择：

| 场景 | 推荐承载 |
|---|---|
| 字段少于等于 5 个，且是轻量确认 | Modal |
| 字段少于等于 8 个，需要保留上下文 | Drawer |
| 字段超过 8 个或存在分组 | 独立页面 |
| 字段超过 20 个或存在流程 | 分步表单页面 |
| 只读详情且内容较少 | Drawer |
| 详情内容复杂、有关联表格 | 独立详情页 |

禁止在 Modal 中承载过长表单。禁止在 Modal 中再打开 Modal。

## 验收说明

本文件只负责布局规则，不再单独维护 P0 gate 列表。

最终验收统一以 `references/checklist.md` 为准：

- 页面框架位置关系
- 多页签和左侧导航选中态
- 白色内容容器
- 页面类型专项检查
