# XFT 页面配方说明

## 目标

页面配方用于固定内容区的默认装配顺序，降低 AI 自由布局的空间。

配方不是整页 HTML，而是页面级装配协议：

```text
页面类型 → 默认区域顺序 → 必选区域 → 可选模块 → 禁用组合 → 校验规则
```

## 核心页面配方

### 1. 基础表格页

默认顺序：

```text
页面头部 → 筛选区 → 表格工具栏 → 表格区 → 分页区
```

可选模块：批量操作区、详情抽屉、空状态、表头设置。

适用：查询、筛选、数据管理、花名册、台账。

禁用：不得把批量操作长期塞进行内；不得省略分页；不得把筛选区和工具栏混为一个无结构区域。

### 2. 可编辑表格页

默认顺序：

```text
页面头部 → 表格工具栏 → 可编辑表格 → 底部保存区
```

适用：批量录入、明细维护、行内编辑。

禁用：不得用于复杂长表单；不得无保存/取消动作。

### 3. 父子表格页

默认顺序：

```text
页面头部 → 左树右表分栏
```

适用：组织、分类、分组、主从数据。

禁用：不得上下堆叠导致关联关系弱化。

### 4. 基础表单页

默认顺序：

```text
页面头部 → 表单卡片 → 底部操作区
```

适用：新建、创建、提交、申请。

禁用：不得把提交按钮放在表单中间；不得无取消动作。

### 5. 分步表单页

默认顺序：

```text
页面头部 → 步骤条 → 当前步骤表单 → 步骤操作区
```

适用：多阶段提交、多步骤申请、流程化录入。

禁用：不得一次性展开全部步骤；不得缺少上一步/下一步。

### 6. 基础详情页

默认顺序：

```text
页面头部 → 摘要区 → 信息分组
```

可选模块：审批流、附件、关联表、操作记录。

适用：单据详情、对象详情、审批详情基础形态。

禁用：不得把所有信息塞进一个超长描述列表。

### 7. 设置页

默认顺序：

```text
页面头部 → 左侧锚点/分组导航 → 右侧设置分组 → 设置项
```

适用：参数配置、权限配置、偏好设置。

禁用：不得把设置项直接做成普通表格；每个设置项必须说明影响。

### 8. 业务首页

默认顺序：

```text
页面头部 → 业务概览 → 栏目/资源入口 → 待办/动态/案例
```

适用：门户、运营首页、工作台。

禁用：不得做成营销落地页；不得只有指标没有路径。

### 9. 审批详情页

默认顺序：

```text
页面头部 → 摘要区 → 审批流 → 业务详情 → 操作记录
```

适用：审批、流程、流转、待办处理。

禁用：审批流不得当普通步骤条使用；必须区分当前节点和历史节点。

### 10. 复杂配置页

默认顺序：

```text
页面头部 → 左侧对象/字段列表 → 右侧配置编辑面板 → 预览/保存
```

适用：筛选项配置、表头设置、公式编辑器、字段配置。

禁用：不得用普通表单承载复杂配置；必须有保存路径。
