# Delivery Checklist

本文件是按需读取的人工复核参考，不是每次生成的强制节点。

只有在以下情况才读取本文件：
- 用户要求人工复核
- 需要排查 Final Check / validator 未覆盖的问题

生成时不需要默认读取本文件。

## P0 - 必须通过

### 1. 路由与页面类型

- [ ] 当前页面类型判断正确：首页 / 表格页 / 提单页 / 详情页 / 编辑页 / 设置页。
- [ ] 页面主体结构与页面类型匹配，没有拿错页面模板。
- [ ] 如果是后台业务页，已使用统一后台框架；如果是明确例外页，未强行套入后台框架。

### 2. 页面框架

- [ ] 页面存在完整 `AppShell`。
- [ ] 顶部导航位于页面最上方，高度 48px，背景渐变为 `#3C8AFF -> #4255FF`。
- [ ] 左侧导航位于顶部导航下方左侧，固定宽度 188px。
- [ ] 多页签位于顶部导航下方、页面内容上方。
- [ ] 页面内容区位于多页签下方。
- [ ] 页面内容没有直接贴在顶部导航下方。
- [ ] `TopNav`、`SideNav`、`WebTabs` 没有被重复生成到 `PageContent` 内部。

### 3. 顶部导航与多页签

- [ ] 顶部导航 tab 选中态为白字加粗 + 字下 2px 白色条，不是背景块。
- [ ] 顶部导航 tab 左右内边距为 24px。
- [ ] 多页签中当前页面有选中态。
- [ ] 多页签至少展示首页和当前页面，且有关闭图标。
- [ ] 多页签右侧包含上一页、下一页、更多按钮区域。
- [ ] 多页签过长时做省略，不换行。
- [ ] 没有用页面内部内容 Tab 替代 `WebTabs`。

### 4. 左侧导航

- [ ] 左侧导航只有一级和二级，没有三级。
- [ ] 一级导航是描边 icon + 文案。
- [ ] 一级导航 icon 使用 `fill="none"` 和 `stroke="currentColor"`。
- [ ] 二级导航仅展示文案，不展示 icon。
- [ ] 当前页面对应的二级导航有选中态。
- [ ] 当前页面所在一级导航同步体现激活状态。
- [ ] 选中态颜色为 `#1966ff`，二级选中背景色为 `#EBF3FF`。
- [ ] 导航项高度、宽度、内边距符合规范：40px / 172px / 8px。
- [ ] 二级导航文案左侧起始对齐到 36px。

### 5. 页面主体

- [ ] 所有业务主体内容都在 `page-content > .page-content-container` 内。
- [ ] 页面主体内容都在白色背景容器中展示。
- [ ] 没有让内容裸露在白色容器之外。
- [ ] 页面结构命名和视觉语义对应真实业务组件，而不是纯 `div` 堆叠伪装。

### 6. 页面类型专项检查

#### 首页

- [ ] 包含概览卡片、快捷入口、待办、趋势/看板、最近访问中的大部分核心模块。
- [ ] 没有把首页做成单一表格页或空欢迎页。

#### 表格页

- [ ] 包含 `PageHeader + FilterBar + TableToolbar + DataTable + Pagination`。
- [ ] 查询条件没有放进 `TopNav`。
- [ ] 没有用卡片列表替代主表格，除非用户明确要求。

#### 提单页

- [ ] 包含标题、表单分组、字段、附件/说明、底部操作区。
- [ ] 字段较多时已分组，不是长列表平铺。
- [ ] 有提交和取消/返回操作。

#### 详情页

- [ ] 包含标题、状态、详情区、关联内容或时间线中的主要部分。
- [ ] 没有默认变成可编辑表单。
- [ ] 有返回或主要操作。

#### 编辑页

- [ ] 结构与提单页相近，但已体现回填状态。
- [ ] 有保存和取消。

#### 设置页

- [ ] 使用 `SettingsSection` / `SettingItem` 这类设置结构。
- [ ] 不是普通表格页。
- [ ] 设置项有标题、说明、状态/控件。

## P1 - 应尽量通过

### 1. 内容真实性

- [ ] 标题、字段名、按钮文案、状态文案都像真实业务，不像占位稿。
- [ ] 没有明显假数据堆砌，没有“字段1 / 字段2 / 功能A / 功能B”这类占位文案。
- [ ] 列表页列名、筛选项、操作项彼此一致，像同一个业务对象。
- [ ] 详情页和编辑页中的状态、字段、操作前后一致。

### 2. 视觉一致性

- [ ] 颜色、圆角、阴影、边框密度符合 `DESIGN.md`。
- [ ] 没有引入随机渐变、玻璃拟态、夸张阴影、装饰性背景。
- [ ] 页面信息密度高但不拥挤。
- [ ] 标题、说明、筛选区、表格、表单、操作区层级清晰。

### 3. 组件正确性

- [ ] 表格页真的像管理列表，不像营销页。
- [ ] 设置页真的像设置中心，不像普通表单页。
- [ ] 提单页和编辑页有差异，不是简单换标题。
- [ ] 详情页不是“禁用态表单”伪装出来的。

### 4. 交互与状态

- [ ] 主操作位置明确，通常位于 `PageHeader` 右侧或 `ActionBar`。
- [ ] 空状态、加载态、禁用态、状态标签等基础状态有体现。
- [ ] 页签、导航、按钮的选中/悬浮/禁用关系不冲突。

## 增量检查

### 范围检查

- [ ] 是否先判断了 `Full Page / Page Overlay / Component State`？
- [ ] 弹窗 / 抽屉 / 确认框需求是否正确判定为 `Page Overlay`？

### Shell 检查

- [ ] 标准后台页面是否使用 `admin-side-shell`？
- [ ] 独立弹窗、局部组件、无导航页是否使用 `blank-shell`？
- [ ] 是否只选择了一个正确的基础 shell，而不是混用多个壳子？

### 导航检查

- [ ] 是否替换了 `TopNav / SideNav / WebTabs / Breadcrumb` 文案？
- [ ] 当前页面是否在左侧导航和多页签中同时有正确选中态？
- [ ] 是否没有把页面内部内容 tabs 误当成系统级 `WebTabs`？

### 内容容器检查

- [ ] 是否存在 `page-content-container`？
- [ ] page block 是否确实放入 `PAGE_CONTENT_SLOT`？
- [ ] 是否没有把完整 shell 结构复制进 page block？

### Overlay 检查

- [ ] 弹窗 / 抽屉需求是否没有误扩成整页？
- [ ] `overlay-root` 是否位于顶层，而不是嵌入 `page-content` 或滚动容器？
- [ ] 页面内 overlay 是否只保留必要宿主背景，没有重构完整宿主页面？

### 组件复用检查

- [ ] 是否复用了已有 class？
- [ ] 是否没有为按钮、表格、状态标签、表单控件临时发明同类样式？

### 输出检查

- [ ] 是否输出完整单文件 HTML？
- [ ] 输出文件是否位于 `examples/` 根目录，且符合 `{slug}-{YYYY-MM-DD}-v<N>.html` 命名格式？
- [ ] 是否没有输出到 `examples/archive/` 或 skill 包外部？
- [ ] `XFT_ROUTE` 是否紧跟 `<!DOCTYPE html>` 之后？
- [ ] 第一个 `<style>` 是否确实为 `tokens.css` 原文？
- [ ] style 块顺序是否为：tokens.css → shell 原 style → 组件 CSS（来自 components.html） → 页面级新增 CSS？
- [ ] CSS 是否全部内联，无外链 stylesheet？

### 示例复制检查

- [ ] 是否没有复制 `examples/` 中的业务字段、页面标题、示例数据、品牌信息和业务流程？
- [ ] 是否没有读取或引用 `examples/archive/` 下的任何文件？archive 目录仅供人工历史对比，不得作为生成依据。

## 复审闭环

P0 和 P1 检查之后，还必须做一次简短复审。至少回答以下四个问题：

1. 这个页面类型判断对了吗？
2. 页面主体是不是放在了正确的后台壳子里？
3. 组件结构是不是和业务语义一致？
4. 如果把标题和业务名遮住，这个页面还像一个真实后台页面吗？

如果以上任一问题答案是否定的，必须回退修正后再输出。
